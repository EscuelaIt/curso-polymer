¿Qué es Polymer?

Una biblioteca

Conjunto de características para la creación de custom elements.

Diseñada para hacer más fácil y rápido el desarrollo de componentes.

Conceptos

Web components

Enstándares que te permiten construir nuevos componentes.

Mucho código y mucho trabajo.

Biblioteca Polymer

Sintaxis declarativa, menos código y más sencillo.

Añade características: plantillas, enlace de datos bidireccional y observación de propiedades.

Custom elements

Elementos construidos con Polymer.

Características

  • Instanciación usando un constructor o document.createElement.
  • Configuración usando atributos o propiedades.
  • Contiene un DOM local dentro de cada instancia.
  • Responde a cambios en propiedades y atributos.
  • Estilado con los internos por defecto o desde fuera.
  • Responde a métodos que lo manipulan inernamente.

Ejemplo

<dom-module id="elemento-saludar">
  <template>
    <style>/* Reglas CSS para tu elemento */</style>
    <!-- DOM local para tu elemento y data binding -->
    <div>{{saludo}}</div>
  </template>
  <script>
    // Registro del elemento
    Polymer({
      is: "elemento-saludar",
      // Añadir propiedades y métodos al prototipo del elemento
      properties: {
        // Declarar propiedades para la API pública del elemento
        saludo: {
          type: String,
          value: "¡Hola!"
        }
      }
    });
  </script>
</dom-module>

Registrar un componente

Función Polymer

  • Parámetro: Prototipo del componente
    • Propuedad is obligatoria
    • Nombre del componente debe contener un guión (-)
  • Devuelve: Constructor del componente
// Registrar un elemento
var MiSaludo = Polymer({is: 'elemento-saludar'});

// Crear una instancia con createElement:
var el1 = document.createElement('elemento-saludar');

// ... o con el constructor:
var el2 = new MiSaludo();

factoryImpl

Constructor personalizado

  • Solo se llama usando el constructor.
  • Se llama después de haber inicializado el elemento.
var MiSaludo = Polymer({
  is: 'elemento-saludar',
  // Constructor personalizado:
  factoryImpl: function(saludo, redes) {
    this.saludo = saludo;
    this.configurarRedes(redes);
  },
  configurarRedes: function(redes){
    …
  }
});
var el = new MiSaludo('¡Hola!', ['twitter', 'facebook', 'gPlus']);

Extender elementos HTML nativos

  • Solo nativos (input, buton, etc..).
  • Elementos personalizados en futuras versiones
  • Se llaman: elementos personalizados de extensión de tipo (type extension custom elements)
var MyInput = Polymer({
  is: 'my-input',
  // Extiende el elemento input
  extends: 'input'
});

Ciclo de vida

Hooks

Implementa las funciones estandar acortadas

  • created en lugar de createdCallback
  • attached en lugar de attachedCallback
  • detached en lugar de detachedCallback
  • attributeChanged en lugar de attributeChangedCallback

Añade una extra, ready, que se lanza cuando Polymer ha terminado de crear e inicializar el DOM local

Orden de inicialización

  1. created callback
  2. local DOM initialized
  3. ready callback
  4. factoryImpl callback
  5. attached callback
